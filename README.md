Here’s a **tutorial-style guide** on useful built-in tools and commands in Windows Server that system administrators often use, similar to `winver`:

---

## ✅ **1. winver**
- **Purpose:** Displays Windows version and build information.
- **How to run:** Press `Win + R`, type `winver`, and hit Enter.

---

## ✅ **2. msinfo32 (System Information)**
- **Purpose:** Shows detailed hardware and software configuration.
- **How to run:** Press `Win + R`, type `msinfo32`, and hit Enter.

---

## ✅ **3. dxdiag (DirectX Diagnostic Tool)**
- **Purpose:** Displays system specs, graphics, and sound details.
- **How to run:** Press `Win + R`, type `dxdiag`, and hit Enter.

---

## ✅ **4. taskmgr (Task Manager)**
- **Purpose:** Monitors CPU, memory, disk, and network usage.
- **How to run:** Press `Ctrl + Shift + Esc` or type `taskmgr` in Run.

---

## ✅ **5. perfmon (Performance Monitor)**
- **Purpose:** Tracks system performance counters for CPU, memory, disk, and network.
- **How to run:** Press `Win + R`, type `perfmon`, and hit Enter.

---

## ✅ **6. eventvwr (Event Viewer)**
- **Purpose:** Reviews system, application, and security logs.
- **How to run:** Press `Win + R`, type `eventvwr`, and hit Enter.

---

## ✅ **7. services.msc**
- **Purpose:** Manages Windows services (start, stop, restart).
- **How to run:** Press `Win + R`, type `services.msc`, and hit Enter.

---

## ✅ **8. diskmgmt.msc (Disk Management)**
- **Purpose:** Manages partitions and volumes.
- **How to run:** Press `Win + R`, type `diskmgmt.msc`, and hit Enter.

---

## ✅ **9. gpedit.msc (Group Policy Editor)**
- **Purpose:** Configures local group policies.
- **How to run:** Press `Win + R`, type `gpedit.msc`, and hit Enter.

---

## ✅ **10. sconfig (Server Configuration)**
- **Purpose:** Configures core server settings (network, updates, domain).
- **How to run:** Open Command Prompt and type `sconfig`.

---

## ✅ **11. PowerShell Cmdlets**
- Examples:
  - `Get-ComputerInfo` → Detailed system info.
  - `Get-Service` → Lists all services.
  - `Get-EventLog` → Retrieves event logs.
